Flexibilidade dos Parametros

function soma (a,b){
return a+b;
}

Nesse caso, é esperado que passemos dois parametros para a função soma.

Se usarmos a função: soma(7,7) ele retornará o número 14. O previsto.

Mas caso passemos mais parametros: soma(7,7,2,3). Ele retornará apenas o 14 e descartará os outros dois números (2,3)

Mas caso passemos parametros a menos: soma(7). Ele interpretará: 7 + undefined = not a number (NaN)

E caso não passemos nenhum valor: soma() ele retorna um Not a Number(NaN)

Uma situação distinta pode ser usada para resolver o caso de mandar só um 1 número em uma função que deveria receber 2 números: fazer os parâmetros (a=0, b=0), o que pod ser vista na função: "soma_diferente", caso seja passada um só numero como exemplo o 7 ele retorna o 7, mas caso seja passado 7 e 3, ele retornará 10. Tudo isso depende da forma que o desenvolvedor deseja resolver o casos exceções.

Mesmo sem ter passado parâmetros, com o uso do arguments podemos recuperar os dados informados e fazer a operação requisitada, sendo um objeto. Essa técnica é muito usada no ECMA de 2009. No ECMA mais recente temos o Spread/Rest
